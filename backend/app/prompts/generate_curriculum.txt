STRUCTURE RULES:
1. Generate exactly {duration_months} months with exactly 4 weeks each ({total_weeks} weeks total)
2. Week numbers must be sequential from 1 to {total_weeks}
3. Last month MUST be a capstone project + job preparation (portfolio, resume, interview prep)
4. Weekly load MUST be calibrated to the "{level}" level (see LEVEL LOAD below)

LEVEL LOAD — do not deviate from these ranges for the "{level}" level:
- beginner:     10–12 hours/week, 3 focus items, 2 deliverables, 3 resources, 3–4 checklist items
- intermediate: 14–16 hours/week, 4 focus items, 2–3 deliverables, 3 resources, 5 checklist items
- advanced:     18–22 hours/week, 5 focus items, 3 deliverables, 4 resources, 6 checklist items
Rationale: beginners need breathing room to absorb fundamentals; advanced learners can sustain denser, research-heavy weeks. A 3-month beginner plan MUST NOT have the same weekly load as a 3-month advanced plan.

BLOOM'S TAXONOMY — progressive cognitive levels by month position:
- First 25% of months: Remember/Understand (define, explain, identify core concepts)
- Middle 50%: Apply/Analyze (implement, build, debug, compare approaches)
- Final 25%: Evaluate/Create (design systems, justify decisions, build end-to-end project)

CHECKLIST QUALITY — every item MUST:
- Start with an ACTION VERB: Implement, Build, Train, Deploy, Fine-tune, Write, Benchmark, Debug, Compare, Design, Submit, Test, Evaluate, Document, Optimize, Configure, Create
- NEVER start with: Understand, Learn, Know, Study, Explore, Review, Read about, Familiarize, Get familiar, Look into
- Be independently verifiable (could be checked in a GitHub PR or notebook output)
- At least 2 of 5 items per week MUST include measurable criteria with numbers (e.g. "Train a CNN achieving >90% accuracy on CIFAR-10", "Build an API handling 100 req/s", "Write tests with >80% coverage")
- The other items should reference concrete artifacts: "Build a notebook", "Deploy to HuggingFace Spaces", "Submit PR with pipeline code"

RESOURCE QUALITY:
- Resources must be REAL, FREE, and currently accessible
- Each week should have a MIX of resource types:
  * 1 structured learning (course/tutorial from: coursera.org, fast.ai, deeplearning.ai, d2l.ai, cs231n.stanford.edu, youtube.com)
  * 1 official documentation or paper (pytorch.org, tensorflow.org, huggingface.co/docs, arxiv.org, scikit-learn.org)
  * 1 hands-on practice (kaggle.com, github.com repos, colab notebooks, paperswithcode.com)
- URLs must be full https:// links you are confident exist as of 2026

THEORY-PRACTICE BALANCE:
- Weeks 1-2 of each month: 60% theory (concepts, lectures, papers), 40% practice
- Weeks 3-4 of each month: 40% theory, 60% practice (build, experiment, deploy)
- Every month must include at least one project deliverable that produces a GitHub-pushable artifact

DELIVERABLE QUALITY:
- Each deliverable must be a concrete artifact: notebook, trained model, API endpoint, report, deployed app
- At least 2 deliverables per course should be portfolio-worthy (real dataset, real problem, shareable)
- Deliverables should build on each other across months (cumulative learning, not isolated exercises)

COMPLETENESS — ensure the curriculum covers:
- Core theory and fundamentals for {topic}
- Practical implementation with modern frameworks (PyTorch preferred, TensorFlow acceptable)
- Model evaluation, testing, and debugging (not just training)
- Deployment/serving (even basic: Flask API, Gradio demo, HuggingFace Space)
- Ethics and responsible AI considerations (bias, fairness, interpretability) — at least 1 week
- Data quality, preprocessing, and feature engineering

LEVEL CALIBRATION for {level}:
- beginner: start from absolute basics, assume Python knowledge only, no ML background
- intermediate: assume ML fundamentals (regression, classification, neural networks), focus on depth
- advanced: assume strong ML experience, focus on research papers, state-of-the-art, system design
